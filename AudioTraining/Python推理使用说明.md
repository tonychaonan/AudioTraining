# Python推理集成说明

## 概述

训练标注软件现在支持使用Python进行推理验证，提供更简单、更可靠的推理体验。

## 架构设计

```
训练标注软件（AudioTraining）
├── 训练：Python (train_wrapper.py) ✅
├── 验证推理：Python (inference_wrapper.py) ✅ 新增
└── UI显示：C# (Form1.cs) ✅

量测软件（Audio900）
└── 推理：C# (YoloOBBInference.cs) ✅ 保留
```

## 优势

### Python推理（训练软件）
- ✅ **代码简洁**：3行代码完成推理
- ✅ **调试容易**：Ultralytics封装完善，错误信息清晰
- ✅ **无需关心Sigmoid/NMS**：库自动处理
- ✅ **与训练一致**：同一套代码，不会出现训练/推理不匹配
- ✅ **自动处理预处理和后处理**：图像缩放、填充、坐标转换全自动

### C#推理（量测软件）
- ✅ **无需Python环境**：独立运行
- ✅ **性能最优**：直接内存操作
- ✅ **UI集成简单**：直接在PictureBox绘制

## 使用方法

### 1. 训练模型

在"模型配置"标签页：
1. 选择模型类型（标准检测 或 旋转检测 OBB）
2. 设置训练参数
3. 点击"开始训练"

训练完成后会自动导出ONNX模型和保留.pt模型。

### 2. 加载模型进行验证

在"验证推理"标签页：
1. 点击"加载模型"
2. 选择训练生成的ONNX或.pt文件
   - **推荐使用.pt文件**：Python推理会自动查找对应的.pt文件
   - 如果只有ONNX，也可以使用（但.pt更准确）
3. 选择测试图片
4. 查看推理结果

### 3. 推理方式

**当前版本自动使用Python推理**，显示信息中会标注：
```
推理方式: Python (Ultralytics)
检测数量: X
最高置信度: 0.xxxx
```

## 文件说明

### inference_wrapper.py
Python推理脚本，负责：
- 加载YOLO模型（.pt或.onnx）
- 执行推理
- 返回JSON格式结果

### Form1.cs - PredictWithPython方法
C#调用Python推理的桥接方法：
- 启动Python进程
- 传递参数（模型路径、图片路径、置信度阈值）
- 解析JSON结果
- 转换为C#对象

### Form1.cs - RunValidation方法
统一的验证推理方法：
- 调用Python推理
- 绘制检测框（支持旋转框和矩形）
- 显示结果统计

## 故障排查

### 问题1：Python推理无输出
**原因**：Python环境未配置或ultralytics未安装

**解决**：
```bash
pip install ultralytics
```

### 问题2：找不到模型文件
**原因**：模型路径错误

**解决**：
- 确保训练完成后生成了.pt或.onnx文件
- 检查模型路径是否正确

### 问题3：推理结果为空
**原因**：
1. 模型训练失败（输出全是0）
2. 置信度阈值太高
3. 图片中确实没有目标

**解决**：
1. 检查训练日志，确认训练成功
2. 降低置信度阈值（默认0.25）
3. 使用训练集中的图片测试

## 与量测软件的集成

量测软件（Audio900）继续使用C#推理：
1. 导出ONNX模型
2. 在量测软件中加载ONNX
3. 使用YoloOBBInference.cs进行推理

**验证一致性**：
- 先在训练软件中用Python推理验证模型正确性
- 再在量测软件中用C#推理
- 对比结果，确保一致

## 性能对比

| 指标 | Python推理 | C#推理 |
|------|-----------|--------|
| 代码复杂度 | 低 | 高 |
| 调试难度 | 简单 | 困难 |
| 推理速度 | 中等（进程通信开销） | 快 |
| 维护成本 | 低（跟随Ultralytics更新） | 高（需手动同步） |
| 适用场景 | 训练验证、开发调试 | 生产部署、实时推理 |

## 开发建议

1. **训练阶段**：使用Python推理验证模型
2. **部署阶段**：使用C#推理（量测软件）
3. **问题定位**：
   - Python推理正常 + C#推理异常 → C#代码问题
   - Python推理异常 → 模型训练问题

## 技术细节

### JSON通信格式

Python输出：
```json
{
  "success": true,
  "count": 2,
  "predictions": [
    {
      "type": "obb",
      "class_id": 0,
      "confidence": 0.95,
      "points": [x1, y1, x2, y2, x3, y3, x4, y4]
    }
  ]
}
```

C#解析后转换为`YoloOBBPrediction`对象。

### 坐标系统

- Python返回的坐标是原图坐标系
- 无需手动处理缩放和填充
- 直接用于绘制
